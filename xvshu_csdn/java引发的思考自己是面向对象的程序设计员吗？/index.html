java引发的思考自己是面向对象的程序设计员吗？

        这几天学习java，降到面向对象和面向过程的程序设计思想，有一句话：“你认为用了类和对象，就是面向对象的设计吗？”。这句话，我们都精心思考思考，我们用了类，将方法归类，这就是面向对象了吗？

        答案是“NO”，面向对象的核心思想是“抽象”，我以前的程序设计，细细想想，算是面向对象的入门吧，因为我们的程序在很大程度上，抽象做得并不好！举个例子：


例：设计某数据库表的在页面的显示


设计思想1：

        为每个表建立每个表的类，页面，配置好字段和显示格式。

        图示：


        


设计思想2：

        将相似表归类，，进行总结，抽象为类，建立每类表的显示页面，配置字段和显示格式。

        图示：




设计思想3：

        将表抽象为类，字段是其中一个属性，属性是个集合，随读随取，建立一个页面，将显示封装在页面层。

        图示：




       我们对比这几个设计思想的差异：

相同点：

        都用到了类和对象

不同点：

        思想2和思想3都进行了总结，但是思想1没有进行总结

        思想3明显比思想2总结得深入


        通过对比，如果要我们选，我们会选哪种思想作为面向对对象的设计思想呢？如果你好不犹豫选择了3，请问，在实际的开发中是不是1的情况最多，2的情况是稍微有点，3的情况基本没有呢？这就是我今天想说的那个问题，用了类和对象就是面向对象吗？“NO”这仅仅是表象，面向对象重要的不仅仅是使用类和对象，而是一种思想上的改变！

总结：

        1，想到比做到重要

        2，深入挖掘和持续学习有时要相辅相成

        3，程序设计的好坏在人

        4，认为没有什么是不可能的，相信就能做到


        如果我们经常深入挖掘自己的认识，我们就能更快成长，有很多人抱怨程序员累！那么，我们是不是应该考虑，累是不是类没有用好呢？

        你，还认为自己是面向对象的程序设计员吗？
