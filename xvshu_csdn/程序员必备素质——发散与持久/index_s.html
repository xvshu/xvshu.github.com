程序员必备素质——发散与持久

 

1，  留神您的键盘

 

         作为一个爱学习的程序员，勤奋的笔记一直是一个不错的选择，它帮助我们整理知识，梳理思路，有人用海量的笔记来证明自己的博学！但是，笔记的技巧却没有多少人注意，尤其是在信息时代，笔记的软件充斥着我们的生活，强大如office也有一个强大的one note！在键盘上飞舞的双手，分分钟能洋洋洒洒数千字！

         但是，笔记就是尽全尽详就ok了吗？看到罗列如山的文字，不知道有多少人再也不看他一眼！但是不这样去做笔记又能怎么样呢！其实我们可以这样。笔记，在古时是两种文体，这两种文体，分指散文和韵文，这两种文体都是短小精悍的文体！在现代人做笔记却脱离了这种原则，笔记的初衷就是，短小精悍！




         怎么做到短小精悍呢？无非是字数缩小！缩到多少合适呢，感觉自己能掌控就合适，我感觉表达一个意思6个字足矣，就绝不增加一个字，哪怕字母或者符号！这六个字要能让自己再次回想的时候，能够牵着这条线，找到十之七八的内容，我更愿意叫他关键字！

         做笔记关键字就足矣，太多的解释和关系让我们变得懒惰，大脑在看到这些的时候不去运动！懒惰的人的笔记一定是做的最全的，恨不得这本书都抄一遍，因为这样他就可以完全不用想什么，看着笔记就可以了！如果没有意识到这一点，和我一起，现在小心自己的键盘，做短小精悍的笔记！

 

2，  小心您的屏幕

 

         我们都有一种心理，叫怀旧，这是好的，这样我们就可以很好的和以前的知识建立联系，但是，不要让自己禁锢在已有的东西里面，我们每天都对着电脑屏幕，重复着一天一天的生活！有规律的生活让我们不善于再去挖掘和创造，这时的我们需要的是激情和不一样的东西！

         如同我们现在学习的数据库，这几天电脑屏幕一直是耿老师的视频和笔记的软件，一天一天一直是这样！我的思维不自觉的就被禁锢了！屏幕上有的，我不一定有，屏幕上没有的，我一定没有！sql是什么不去想，t-sql是什么，不去想，他们的区别，鬼知道！还有什么数据库，和我有什么关系？？这是一个程序员最要命的地方！不再想为什么！我们的思维就停滞不前！



在数据库学习中，我们应该发现这些：

         Sql(Structured Query Language)是一种语言，他可以再程序设计中更有效的操作数据，发展至今它更是一种标准，具有完全不同底层结构的不同数据库系统，可以使用相同的SQL语言作为数据输入与管理的接口！足见sql的强悍！

        T-SQL(Transact Structured Query Language)它是ANSI和ISO SQL 标准的Microsoft SQL Server方言或扩展，中文理解为：SQL SERVER专用标准结构化查询语言增强版。简言之，它是专业性更强，针对SQL Server的语言，是sql的子系！

Microsoft SQL Server它可以这么理解，微软研发，可提供sql服务的数据管理程序，他是一个程序，如qq，msn一般的一个程序！例如他这样的程序还有Oracle, Sybase, Access等等!

         当然，必要的情况是我们一定要了解了解其他的数据库！

         所以，小心你的屏幕，这里的屏幕意指禁锢我们视界和思维的一切事物，不要沉浸于学习而不思进取，学习的发散性一定要体现在过程中，我们不可能“一口吃成胖子”慢慢积累，多想为什么！发散发散自己的思维！这是锻炼出来的！

 

3，  缓缓您的脚步

 

         有一个现象是，有些东西，我们学过了，就认为自己学过了，只要过了这个时间，再也不去看看我们曾经拥有的东西！这不是明智的做法！有些东西，暂时用不到的是可以束之高阁的，但是束之高阁的目的是为了有一天拿下他来的时候，有一种似曾相识的感觉，而上面的情况确实，完全抛弃，再次见到又像全新的一样！累！是唯一的感觉！




         众所周知学习是个循序渐进的过程！

         有的人把焦点放到了“序”上，以为按一定的程序自己就一定能学会，不知道大千世界，自己体会的才是属于自己的！别人总结的是经验和教训！殊不知，经验要了解，教训要认识，道路要自己走，固定的程序是没有的，如老师所讲“便是永远不变的”！

         有的人把焦点放到“渐”上，也自以为跟着走就行！先不说能不能跟上，像这样出来的人会怎么样，我们来这样想！一起去面试，有几个人，一所学校，你知道的，别人都知道，你不知道的，有些人知道！老板会要怎么样的人呢？这是供大于求的关系，老板一定要最便宜的！所以我们就被“贱卖”了！

         聪明的人是四个字都要兼顾到！在前人的基础上，有序地进行某一件事，根据自己的需要进行一定的回顾和调整，做好这些我们才能慢慢前进而不是停滞不前，所以，学习要在适当是时候停下你的脚步！进行简单的回顾和总结！

 

4，  绷紧一根弦

 

         这里的绷紧一根弦是说的贯彻力！懒是大部分人的通病！计划是计划，执行是执行！偶尔的偷懒，还满是借口，我也不例外，也是这样的，执行力不是太理想是我一直犯愁的事情！对此我的建议是多人共同监督！

         首先找的几个志同道合的战友！公开google日历！在不定的时间不定的人互相检查，实行鼓励、惩罚共同进行！力度视团队经济能力，和抗打击能力而行！

 

         最后祝福所有奋斗在一线的程序员！房子会有的，车子会有的，一切都会好起来的！
